# 📝 プロンプトガイドライン

## 1. 回答スタイルの基本ルール

- **言葉**: **短く簡潔**、**専門用語を避け**、**分かりやすい言葉**を使用
- **回答形式**: 質問内容に直接回答

---

## 2. IT 用語リスト作成の要件

JSON リスト作成時、以下の条件をすべて満たす

### 複数データ

- アルファベット順表示

### 単語

- **6 文字以下**
- 最新の IT トレンド用語
- **これまでの会話で提示した単語とは重複させない**

### 説明（韓国語/日本語/英語）

- **30 文字以下**で、簡潔に記述
- 先頭 1 桁は、**単語を連想させる絵文字**にする
- 混同しやすい用語については、その単語を特定するための適切な単語を必ず追加する
- 「〜する方法/技術/システム」のように、**名詞（Method, Technology, System など）で開始**
- 用語と直接関係ない単語は、なるべく使用しない

### 形式

- 以下 JSON フォーマットのように回答

```
# JSONデータの例
  {
    "word": "ACCESS",
    "description_ko": "🔑시스템의 정보나 기능을 이용하는 권한",
    "description_ja": "🔑システムの情報や機能を利用する権利",
    "description_en": "🔑Right to use system data or functions"
  },
```
